 Key Design Decisions Reflected

 - Task IDs: BE{epic}-S{story}-T{task} convention (matches frontend E{epic}-S{story}-T{task})
 - Pydantic schemas match frontend TypeScript types field-for-field — sourced from src/types/*.ts
 - API response wrapper: { data: T, status: "success", timestamp } matching ApiResponse<T>
 - WebSocket format: { type, payload, timestamp, sequenceId } matching WSMessage
 - Sentiment scoring: Weighted ratio formula from binding decisions (NOT Bayesian)
 - Auth: Custom JWT with refresh rotation, three tiers (free/pro/quant)
 - Batch delivery: 5-10s batches for sentiment, 100ms batches for WebSocket
 - Dependency graph included showing epic ordering
