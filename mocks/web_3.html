Near-instant HMR — changes reflect in the browser in under 50ms. ESM-native means no Webpack config debt to maintain, no CRA baggage to fight. Start in milliseconds, not seconds.

STATE
Zustand
+ TanStack Query v5
Client state Server state
Two different problems, two purpose-built tools. Zustand for client state (no boilerplate, no reducers, no provider hell). TanStack Query for server state — handles caching, background refetch, stale-while-revalidate, and deduplication automatically.
NO BOILERPLATE
STYLING
Tailwind CSS v4
+ shadcn/ui
Utility-first Headless Owned code
Utility-first CSS means style lives next to markup — no context switching, no naming things. shadcn/ui gives you headless primitives you own — copy into your repo, customise freely, no version-lock to a component library's design decisions.
FULL CONTROL
ANIMATION
framer-motion
layout + gesture + spring
Layout animation Exit transitions
CSS animations handle simple transitions. framer-motion handles the hard cases: shared layout transitions, exit animations, drag gestures, spring physics. The difference between an interface that "works" and one that genuinely feels good.
PERCEIVED QUALITY
VIRTUALIZATION
react-virtuoso
dynamic list virtualization
Variable row heights DOM recycling
Only render what's visible — recycle DOM nodes as the analyst scrolls. Without this, a feed of 5,000 items renders 5,000 React components. With it, the DOM stays at ~15 nodes regardless of list length. Non-negotiable for a real-time intelligence feed.
PERFORMANCE
GEO
Three.js
+ react-three-fiber
WebGL GPU-instanced R3F
GPU-instanced markers render 10,000+ simultaneous data points at 60fps where SVG or Canvas 2D would collapse under 200. react-three-fiber wraps Three.js in React's component model — declarative scene graphs, reactive updates, no imperative lifecycle management.

ROUTING
react-router-dom
v7 · data router
Code splitting Lazy loading
Routes are lazy-loaded and code-split by default. The analyst's browser only downloads the JavaScript for the route they're on — not the entire application bundle upfront. Faster initial load, faster subsequent navigation.
BUNDLE SPLITTING
VALIDATION
Zod
runtime schema validation
Parse, don't trust Inferred types
TypeScript types exist only at compile time — they vanish at runtime. When data arrives from an external API at 03:41 Zulu, Zod validates its shape against the schema and throws a precise error if anything is wrong. Type inference means one schema definition produces both the runtime validator and the TypeScript type.
